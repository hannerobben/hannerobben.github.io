Logic Tree

C# | ASP.NET Web Forms | Linq | SQL | JavaScript | JQuery | HTML | CSS

2022

In context of employment | Project lead

What?

An interactive library of products and their parts where a product is defined as a 3-level structure: product > subparts > parts (e.g. motorcycle > engine > piston). For the lowest level parts users can access product standards which define the required specifications for that part.

A control panel is available to administrators who can manage products, product structures and product standards. For the product standards, which can be updated, there is a complete version history available including the option to revert to older versions.

Creation of and updates to product standards require confirmation, which results in a multi-layered process of drafting, submission, rejection and implementation involving multiple users.

How?

The web application was developed in several iterations where I presented and demoed the application to the client. I wrote back-end as well as front-end code and I had creative freedom to design parts of the UI and UX.

Challenges?

This project required me to comprehend and implement complex application logic - including database design, data queries and algorithms.

Code efficiency was as especially important on this project as it required complex queries to run on large amounts of data while keeping short loading times.

Note: this project is not publicly available.

Raize

C# | ASP.NET Web Forms | Linq | SQL | JavaScript | JQuery | HTML | CSS | Chart.js

2022

In context of employment | Project lead

What?

An application to raise faults/defects against a product during the production process. Products are defined as a 3-level structure: product > subparts > parts (e.g. motorcycle > engine > piston). The lowest level parts can be selected in a visual way through markers on a product image, after which a fault/defect can be submitted against that specific part.

A control panel is available to administrators who can manage production processes, products, product structures and faults. Through a visual reporting tool using Chart.js, extensive data on faults can be extracted for analysis.

This application is linked to Logic Tree (see application above).

How?

The web application was developed in several iterations where I presented and demoed the application to the client. I worked in close relationship with the client to think of relevant new features and to streamline application processes. I wrote back-end as well as front-end code and I had creative freedom to design parts of the UI and UX.

Challenges?

The application required a reporting tool to extract useful data for which I needed to write complex data queries and algorithms and called for smart database design. This data also had to be visualised in a comprehensive way by using the Chart.js library.

I was given a lot of freedom in terms of defining application logic, suggesting new features, designing UI and UX which required an adequate amount of autonomous decision making and critical thinking.

Note: this project is not publicly available.
